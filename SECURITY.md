# Security Policy

## Supported Versions

This project is in early development. Security reports should target the latest unreleased state until versioned releases begin.

## Reporting A Vulnerability

Please do not open public issues for sensitive security reports.

Use the repository security advisory flow when available, or contact the maintainers through the channel listed in the published repository.

## Security Expectations

- Generated projects must not include real secrets.
- Templates must not hard-code private credentials.
- User input should be treated as untrusted.
- Dependencies should be kept minimal and justified.
