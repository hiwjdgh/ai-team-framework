# Product Overview

## Vision

AI Team Framework enables users to create production-oriented software projects by describing requirements once and letting a coordinated team of AI agents plan, design, implement, verify, and document the result.

## Primary Goal

The framework will eventually be used through an `npx` command.

Example target usage:

```bash
npx ai-team-framework create
```

The CLI should guide the user through project type, requirements, stack preferences, design direction, data model, deployment target, and quality constraints.

## Target Outputs

The framework should support generation of:

- Full web applications
- Homepages and landing pages
- Admin dashboards
- SaaS products
- CRUD systems
- API services
- Documentation sites
- Internal business tools

## Target Users

- Solo builders
- Small teams
- Product owners
- Developers who want structured AI-assisted scaffolding
- Agencies that repeatedly create similar software foundations

## Product Principles

- Structured before creative.
- Reusable before one-off.
- Explicit before implicit.
- Testable before impressive.
- Maintainable before fast-looking.

## Non-Goals For The Foundation Stage

- No production CLI is required yet.
- No runtime agent orchestration code is required yet.
- No package publishing is required yet.
- No third-party dependency decision should be made without implementation context.
